## 原因分析（为什么一致或为什么不一致）

### 1) `TOT_FLUX` 与 `INF_FLX` 的数量关系

可建立的关系：

- `INF_FLX(1) + INF_FLX(3) = 1.78266E+17 + 4.07585E+16 = 2.190245E+17`
- `TOT_FLUX(1) = 2.19024E+17`

两者几乎一致（相对差约 `2.3E-6`）。

同时：

- `TOT_FLUX(3) + TOT_FLUX(5) = 6.59212E+16 + 1.53103E+17 = 2.190242E+17`

这也与总量一致，说明 `TOT_FLUX` 与 `INF_FLX` 在“全局归一化积分通量”层面是自洽的。

### 2) `DETflux_2g` 与 `TOT_FLUX/INF_FLX` 的关系

`DETflux_2g`（第11列）按能群求和得到：

- g1（289栅元求和）=`3.1331476E+16`
- g2（289栅元求和）=`1.00329579E+17`
- 总和=`1.31661055E+17`

与 `TOT_FLUX(1)=2.19024E+17` 比值约为 `0.6011`，并非固定 1。

此外，群占比也明显不同：

- DET: g1/g2 ≈ `23.8% / 76.2%`
- INF: 两群占比 ≈ `81.4% / 18.6%`（按 `INF_FLX` 两群值）

因此，不一致类型属于：

- **统计/归一化口径不一致**（全局积分 vs 探测器统计量）
- **几何口径不一致**（`INF_FLX` 是 infinite geometry/gcu 输出；`DETflux_2g` 是 `dl 1000` 格点探测器输出）
- **可能还存在群顺序与体积权重口径差异**（需要显式核对）

所以三者不能直接逐项相等比较。

## 可执行校验建议（确认 DETflux_2g 来源是否正确）

1. **先做源头链路校验**
   - 在 `A001.sss` 中确认定义：`det flux_2g n de e2g dl 1000`
   - 在 `A001.sss_det0.m` 中确认输出对象：`DETflux_2g`、`DETflux_2gE`
   - 校验 burnup 步一致（res 与 det 文件对应同一计算步）

2. **核对能群边界与群序**
   - `e2g` 边界：`[1E-11, 0.625E-6, 20.0]`
   - 用 `DETflux_2gE` 逐项比对群边界，确认 DET 的 group1/group2 与 `INF_FLX` 的群序映射

3. **避免跨口径直接比较**
   - `INF_FLX/TOT_FLUX` 属于全局/无限介质口径
   - `DETflux_2g` 属于晶格分箱探测器口径
   - 先在同口径下比较（例如增加不带 `dl` 的全局 2群探测器，再与 `INF_FLX` 对照）

4. **做体积权重核验**
   - 对 `DETflux_2g` 使用栅元体积权重：
     - 体积积分：`Φ_g = Σ_i φ_{g,i} V_i`
     - 体积平均：`φ̄_g = Σ_i φ_{g,i}V_i / Σ_i V_i`
   - 再与同定义口径的全局2群结果比较

5. **最终判据**
   - 若“定义一致 + 群序一致 + 归一化口径一致 + 体积权重一致”后仍偏差明显，再检查：
     - 探测器估计器定义（track-length/collision）
     - 输出单位说明段
     - 是否使用了额外过滤器（空间、材料、反应类型）
